前端面试题：TypeScript高级类型与常见陷阱（2026版）

发表于 2026-06-06 分类于 前端

本文整理了 2026 年 TypeScript 面试中高频出现的高级类型、常见陷阱以及一个经典线上 bug 的排查过程。每题附参考来源，适合中高级前端准备面试。

0x00 面试题

1. any、unknown、never 的区别是什么？什么时候用哪个？

  • any：关闭类型检查，兼容所有类型，也会污染后续推导。适合快速原型或渐进迁移时过渡使用。
  • unknown：安全的 “任意类型”。不能直接操作，必须先做类型收窄（typeof / instanceof / 自定义 type guard）才能使用。
  • never：表示永远不会发生的类型。常用于抛出异常的函数返回值、switch-case 中穷尽性检查。

参考：TypeScript 官方文档 — Basic Types / Narrowing

2. type 和 interface 的核心差异？

特性 interface type
声明合并 同名 interface 自动合并 不允许
扩展方式 extends & 交叉类型
联合类型 不支持 支持 type A = B | C
元组/映射类型 不直接支持 支持

选择策略：描述对象形状用 interface（可扩展性更好），需要联合类型、元组、映射类型用 type。

参考：TypeScript Handbook — Interfaces vs Types

3. TypeScript 的协变与逆变，以及 --strictFunctionTypes 的影响

这是一个常被问到但很多人答不上的高阶题。

  • 协变：子类型可以赋值给父类型。返回值是协变的。
  • 逆变：父类型可以赋值给子类型。参数是逆变的（启用 strictFunctionTypes 后）。

考察点：理解类型安全边界，知道什么时候类型推导会出问题。

参考：TypeScript 2.6 Release Notes — strictFunctionTypes

0x02 场景问题：前端大文件上传怎么做？

问题：用户需要上传 2GB 的视频文件，如何保证稳定性？

方案：分片上传 + 断点续传 + 秒传检测。

关键点：

  • 用 Web Worker 或 WASM 计算文件 hash，避免阻塞主线程
  • 并发数控制在 3-6 个，避免占满带宽
  • 加上进度条和暂停/恢复功能提升体验

0x03 小结

这篇文章覆盖了 TypeScript 面试中的三个高频方向——类型系统的深度理解（any/unknown/never、协变逆变、type vs interface），再加上一个真实的 React 闭包 bug 和生产级大文件上传方案。前端面试越来越看重「原理理解 + 实战经验」的组合，不要只背题。

0x03 踩坑记录

语言显示异常

NexT 的语言文件命名是 zh-CN.yml，如果写 zh-Hans 会匹配不到，回退到字母序第一个语言（孟加拉语 bn），导致界面文字乱码。

解决：_config.yml 中 language: zh-CN

GitHub 邮箱隐私保护

书籍推荐

发表于 2019-08-13 更新于 2026-06-06 分类于 fortest

0x00概述

初次推荐书籍，这里我看的书籍也不多，目前也在充实自己中，欢迎大家推荐和自荐~

补充，推荐的书籍，各大商城均有销售，价格起伏不定，请适时下手~

以下按照数据中转的流程推荐书籍~

0x01书单

一、《深入理解Kafka核心设计与实践原理》 Kafka类书籍个人入门推荐5星

厮大著作的书基于Kafka2.0~ 此书基础扎实，适合小白入门，相关慨念颇多，涉及Kafka的架构设计和很多高级应用，读此一本计科了解Kafka的整体架构和理念，建议配合书上搭建源码环境进行学习，效果更佳~

二、《HBase不睡觉书》HBase类书籍个人入门推荐五星

杨曦大佬著作的书，适合小白和有一定基础的工程师入门，书中知识点比较琐碎，但是按照杨大佬的推荐，阅读此书时，需要有选择性的阅读，不适合从头开始看(如果你有基础)。看完确实对Hbase的架构有了一定的认识，还对Hbase的发展有了一定的了解~

三、《大数据之路：阿里巴巴大数据实践》数仓类书籍个人推荐超5星

这本介绍了阿里大数据的最佳实践，适合小白和数仓工程师阅读，书中案例结合阿里遇到的问题进行将结合阐述，在数据模型的设计和相关的平台体系建设都有相应的实践。很多大佬都强烈推荐这一本书,哈哈~
