大数据面试题：数据湖面试全解

发表于 2026-06-07 分类于 大数据

数据湖高级面试全解，覆盖 Iceberg / Hudi / Delta Lake 三大框架的核心原理、选型对比和湖仓一体架构。

0x00 数据湖概念

1. 数据湖和数仓有什么区别？

维度 数据仓库 数据湖
数据质量 Schema-on-Write（先定义后写入） Schema-on-Read（读取时解析）
数据格式 结构化表 结构化 + 半结构 + 非结构化
事务支持 原生 ACID（传统 RDBMS） 需要表格式层（Iceberg/Hudi）
计算引擎 单一（如 Hive/SparkSQL） 多引擎（Spark/Flink/Presto/Trino）
成本 高（需要 ETL 预处理） 低（数据先入湖再处理）

湖仓一体（Lakehouse）：数据湖的存储成本 + 数仓的事务和管理能力。

0x02 Apache Hudi

4. Hudi 的 COW 和 MOR 表类型怎么选？

类型 写入延迟 读取性能 适用场景
COW (Copy-on-Write) 高（每次写入合并文件） 快（读取最新 Parquet） 读多写少，批处理
MOR (Merge-on-Read) 低（增量写日志） 慢（读时合并 Parquet+Avro) 写多读少，近实时

深入思考：

  • 对比 Iceberg：Hudi 的核心差异是增量查询和索引机制（Bloom Filter / HBase Index），适合 CDC 场景。Iceberg 更适合分析场景的多引擎互操作
