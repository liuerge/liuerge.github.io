大数据面试题：Hive & SQL 面试全解

发表于 2026-06-07 分类于 大数据

Hive 与 SQL 高级面试全解，覆盖分区/分桶、HiveSQL 高级写法、数据倾斜治理、存储格式和性能调优。

0x00 核心概念

1. Hive 分区和分桶的区别？

维度 分区 (Partition) 分桶 (Bucket)
物理存储 独立目录 dt=2026-06-07/ 独立文件 000000_0, 000001_0...
字段类型 通常是日期、地区等低基数字段 通常是用户 ID 等高基数字段
数量上限 数万（太多会 NameNode 压力大） 与数据量和分桶数成正比
适用查询 WHERE dt = '2026-06-07' JOIN on user_id / 抽样

深入思考：

  • 生产经验：分区不是越多越好。单表分区超过 10 万个会导致 Metastore 查询变慢和 NameNode 内存压力。按天分区 + 生命周期管理是常规做法

2. Hive 数据倾斜的产生原因和解决方案？

产生原因：某个 key 的数据量远大于其他 key（如 null 值、热点用户）

解决方案：

方案1：两阶段聚合（适合 Group By 倾斜）

方案2：Map Join（适合小表 Join 大表）

方案3：倾斜 key 分离

5. ORC、Parquet、TextFile 怎么选？

格式 压缩率 查询速度 适用场景
ORC 最高（列式+索引） 最快（内置索引） Hive 主力，适合分析查询
Parquet 高 快 跨平台（Spark/Flink/Presto 通用）
TextFile 无 慢 原始数据落盘、临时表
Avro 中 中 Schema 演化频繁的场景

ORC 加速手段：

  • Predicate Pushdown：orc.bloom.filter.columns = order_id，在读取时跳过不匹配的 Stripe
  • Vectorization：SET hive.vectorized.execution.enabled=true，批量处理 1024 行
