7. 如何编写带 trap 信号处理的 Shell 脚本？

标准答案：

trap 用于捕获信号并执行清理逻辑，是生产脚本的标配：

深入思考：

  • 信号差异：EXIT 在所有退出路径触发（正常 exit、信号终止、set -e 检测到错误）。INT（Ctrl+C）、TERM（kill 默认信号）需要单独 trap
  • 锁竞争：使用文件描述符（exec 200>）配合 flock 比传统的 [ -f lock ] && exit 更可靠——前者在进程崩溃时内核自动释放锁，后者会留下孤儿锁文件
  • 生产经验：trap 中慎用 exit，嵌套 exit 可能导致 trap 再次触发。用 kill -9 $$ 会跳过 trap，用于最后的保底强制退出

参考来源：Bash Manual, 生产运维实践

8. Shell 如何实现并行任务处理？（xargs -P / GNU parallel / wait 管理）

标准答案：

三种并行方案，按复杂度递增：

深入思考：

  • xargs -P 陷阱：xargs -P 不保证输出顺序——两个并行进程的 stdout 可能交错。用 --output (GNU parallel) 或重定向到独立文件解决
  • 错误处理：GNU parallel 的 --halt soon,fail=1 可在首个任务失败后停止新任务。wait 方案需手动收集每个后台 pid 的退出状态
  • 生产经验：GNU parallel 支持 --load（CPU 负载阈值）和 --memfree（内存阈值），避免打爆生产服务器。如 parallel -j 0 --load 80% 是自适应并发

参考来源：GNU parallel manual, Bash Cookbook

9. 生产环境 JSON 处理实战（jq 高级用法）

标准答案：

jq 是生产脚本处理 JSON 的事实标准，远胜 grep/sed/python -c：

深入思考：

  • jq vs python -c：jq 是领域特定语言，处理 JSON 比 Python 一行流更简洁，且启动开销远小于 Python（~2ms vs ~50ms）。但复杂逻辑（如网络调用、数据库查询）仍需 Python
  • 大文件处理：jq 是流式处理（streaming parser），可以处理 GB 级 JSON。用 jq -n 'inputs' file 而非 jq '.' file 可以边读边处理
  • 生产经验：jq -r 输出 raw string（去引号），适合管道传递。jq -c 输出紧凑 JSON（一行一条），适合 while read 逐行处理。jq -e 设置退出码（匹配不到时非零）

参考来源：jq manual, 生产运维实践

10. 如何编写进程守护（Daemon Wrapper）脚本？

标准答案：

守护脚本的核心：监控目标进程存活，异常退出时自动重启，同时防止重启风暴：

深入思考：

  • 重启风暴防护：指数退避（1s → 2s → 4s → 8s… → 60s）配合最大重试次数，防止 bug 导致无限重启循环。StackStorm/Supervisor 等成熟方案也采用类似策略
  • 信号转发：守护脚本收到 SIGTERM 时应转发给子进程（trap 'kill -TERM $CHILD_PID' TERM），否则 kill 守护脚本时子进程变成孤儿
  • vs systemd：systemd 的 Restart=on-failure + RestartSec=5s + StartLimitBurst=5 提供了原生的进程守护能力，生产环境应优先用 systemd 而非自定义守护脚本

参考来源：Unix Daemon 编程实践, systemd.service(5)

以下三个案例来自真实生产环境：

案例 1：Nginx 访问日志 P99 延迟分析

案例 2：多文件日志时间线合并（sed 预处理 + awk 合并）

案例 3：awk 实现滑动窗口异常检测

深入思考：

  • awk 内存管理：大型文件（10GB+）的 END 块可能内存溢出。因为 awk 在 END 块执行前将所有数组和变量保留在内存中。解决方案：用 awk '{...}' file | sort | uniq -c | sort -rn 管道分流（sort 使用磁盘临时文件）
  • sed 流式处理：sed 是逐行处理的，不占用额外内存，适合流式管道。但 sed -i 实际上是先写临时文件再 rename，大文件的 -i 会占用双倍磁盘空间
  • 生产经验：gawk 的 @include 机制可以复用公共函数库。如 @include "json" 配合 gawk-json 扩展解析 JSON 日志，比 jq 更适合流式处理

参考来源：gawk manual, The AWK Programming Language (Aho/Kernighan/Weinberger)
